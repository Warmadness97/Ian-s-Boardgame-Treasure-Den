# 桌遊資料庫（共用版・可部署到 GitHub Pages）

一個可以多人共用、即時同步的桌遊收藏網站。可以上傳/刪除桌遊、顯示圖片、類型、
語言版本、遊玩人數與時間，並支援搜尋與篩選。任何有連結的人都能一起維護同一份清單。

因為 GitHub Pages 只能放靜態網頁（沒有自己的後端伺服器），這個版本改用
**Firebase Firestore**（Google 提供的免費雲端資料庫）來儲存資料，
讓所有人打開同一個網址時看到、編輯的都是同一份清單。

---

## 一、建立你自己的 Firebase 專案（約 5 分鐘，免費）

1. 前往 https://console.firebase.google.com/ ，用 Google 帳號登入。
2. 點「新增專案」，輸入專案名稱（例如 `my-boardgames`），一路下一步建立完成。
3. 進入專案後，點左側選單的 **建構 (Build) > Firestore Database**，
   點「建立資料庫」。位置選離你近的（例如 `asia-east1`），
   模式先選 **「以測試模式啟動」**（之後會在下面設定更明確的權限）。
4. 回到專案總覽頁，點網頁圖示 `</>` 新增一個「網頁應用程式」，
   幫它取個暱稱（例如 `web`），**不需要**勾選 Firebase Hosting。
5. 建立後畫面會顯示一段 `firebaseConfig = {...}` 的設定物件，把裡面的內容
   複製起來。

## 二、填入設定值

打開 `firebase-config.js`，把裡面的內容換成你剛剛複製的設定，例如：

```js
export const firebaseConfig = {
  apiKey: "AIzaSy...",
  authDomain: "my-boardgames.firebaseapp.com",
  projectId: "my-boardgames",
  storageBucket: "my-boardgames.appspot.com",
  messagingSenderId: "123456789",
  appId: "1:123456789:web:abcdef123456"
};
```

## 三、啟用 Google 登入（Firebase Authentication）

現在網站已經加上登入機制：**任何人都能瀏覽、搜尋**桌遊清單，
但只有**登入的人**才能新增、編輯、刪除桌遊或修改標題。

1. Firebase Console 左側選單點 **建構 (Build) > Authentication**，點「開始使用」。
2. 在「Sign-in method」分頁，點 **Google**，右上角開關打開，
   選一個支援電子郵件（通常是你自己的信箱）當作專案的公開名稱來源，儲存。
3. 點上方分頁 **Settings > 已授權網域 (Authorized domains)**，
   按「新增網域」，把你的 GitHub Pages 網域加進去，例如：
   `你的帳號.github.io`
   （`localhost` 和 `你的專案.firebaseapp.com` 預設就有，不用理它）。

完成後，網站右上角會出現「使用 Google 登入」按鈕，點下去會跳出 Google 帳號
選擇視窗，登入後畫面右上角會顯示你的大頭貼、名稱與登出按鈕。

## 四、設定 Firestore 安全規則（重要）

在 Firebase Console 左側 **Firestore Database > 規則 (Rules)**，貼上以下規則
並「發布」。這會讓**任何人（包含未登入者）都能讀取**（瀏覽、搜尋清單），
但**只有你自己的信箱**才能新增、編輯、刪除：

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /games/{gameId} {
      allow read: if true;
      allow write: if request.auth != null
                   && request.auth.token.email == 'ianlin1997@gmail.com';
    }
    match /meta/{docId} {
      allow read: if true;
      allow write: if request.auth != null
                   && request.auth.token.email == 'ianlin1997@gmail.com';
    }
    match /photos/{photoId} {
      allow read: if true;
      allow write: if request.auth != null
                   && request.auth.token.email == 'ianlin1997@gmail.com';
    }
  }
}
```

> **這次更新新增了 `photos` 集合**（活動照片功能），如果你的 Firestore
> 規則是之前設定的，記得到 Firebase Console → Firestore Database →
> Rules，把上面這份新規則貼上去並「發布」，否則活動照片頁面會顯示連線
> 失敗（因為還沒有對應的讀寫權限）。

> 測試模式預設 30 天後會自動關閉權限，記得手動改成上面的規則，
> 網站才會長期可用。其他人登入 Google 帳號後，畫面右上角一樣會顯示他們的
> 大頭貼與名稱，但因為信箱不是 `ianlin1997@gmail.com`，看不到新增/編輯/
> 刪除按鈕，就算用瀏覽器工具硬送出寫入請求，也會被這條規則擋下來。
> 如果之後想多加一個信任的朋友一起管理，把上面兩處
> `request.auth.token.email == '...'` 改成
> `request.auth.token.email in ['ianlin1997@gmail.com', '朋友的信箱@gmail.com']`
> 即可，跟我說一聲我也可以幫你改。

## 五、上傳到 GitHub

1. 在 GitHub 建立一個新的 repository（公開或私人皆可，公開才能用免費的
   GitHub Pages）。
2. 把這個資料夾內的檔案（`index.html`、`styles.css`、`app.js`、
   `firebase-config.js`）全部上傳到 repo 裡（可以直接在網頁上拖曳上傳，
   或用 `git push`）。

## 六、開啟 GitHub Pages

1. 進入 repo 的 **Settings > Pages**。
2. 「Source」選擇 `Deploy from a branch`，Branch 選 `main`，資料夾選 `/ (root)`，
   儲存。
3. 等 1–2 分鐘後，頁面會顯示網址，格式類似：
   `https://你的帳號.github.io/repo名稱/`
4. 打開這個網址，就是你的共用桌遊資料庫了，之後把網址分享給朋友即可一起使用。

## 功能說明

- **首頁（新）**：網站現在分成「首頁」跟「遊戲庫」兩個畫面。首頁在橫幅
  下方，由上到下依序是：
  - **今日推薦**：預設每天自動從收藏裡挑一款桌遊當作精選（依日期決定，
    同一天大家看到的都一樣，換日就會自動換一款），用大卡片顯示圖片、名稱、
    類型、難度跟簡介，點卡片或「查看詳情」會打開完整放大檢視。管理者可以
    點右上角「管理」按鈕，指定固定顯示某一款桌遊（不再自動輪替），要改回
    自動輪替也可以隨時切換回「自動（依日期輪替）」。
  - **推薦與精選**：管理者可以點鉛筆寫一段推薦內容（支援換行），分享最近
    喜歡的桌遊或聚會心得，所有人都看得到；沒有內容時訪客不會看到這個區塊。
  - **進入遊戲庫**按鈕，點下去才會進入原本的搜尋/瀏覽畫面；遊戲庫畫面
    左上角也有「回首頁」按鈕可以隨時返回。
  - **遊戲系列介紹**：自動列出目前收藏裡出現過的所有「系列」，格式跟下面
    的桌遊類型介紹一樣——每個系列用卡片顯示、標出符合的款數，管理者可以點
    鉛筆寫簡介，點卡片本身會直接進入遊戲庫並套用這個系列的篩選條件。
  - **桌遊類型介紹**：自動列出目前收藏裡出現過的所有「類型」標籤，每個
    類型都用卡片顯示，卡片上會標出目前有幾款桌遊符合這個類型；管理者可以
    點鉛筆圖示，用彈出視窗輸入這個類型的簡單說明（例如「合作」類型可以寫
    「大家一起對抗遊戲，成功或失敗都一起承擔」）。點卡片本身（非鉛筆處）
    會直接進入遊戲庫，並自動套用這個類型的篩選條件。
- **系列堆疊卡片（新）**：在遊戲庫瀏覽（沒有特別勾選「系列」篩選）時，
  同一系列的桌遊會像一疊卡牌一樣合併成一張卡片顯示，最上面顯示的是這個
  系列的封面——預設規則是優先顯示「主遊戲」，如果系列裡都是主遊戲、沒有
  擴充，就用最早上傳的那一款當封面；管理者也可以點卡片上的「選擇封面」
  按鈕，手動指定要顯示哪一款。卡片上會有一行「📚 這個系列共有 N 款作品」
  的說明文字（獨立一行顯示，不會擠進系列標籤裡造成標籤過長換行）。
- **卡片圖片尺寸統一**：所有桌遊卡片的縮圖區改用固定高度顯示，不論原始
  圖片比例為何，畫面上的縮圖大小都會保持一致。
- **卡片整體尺寸統一**：每張卡片的整體高度也固定，不會因為簡介長短、
  標籤數量不同而大小不一，畫面看起來更整齊。名稱、擴充來源等文字太長時
  會截斷顯示「...」，類型標籤太多時比照語言版本的做法，只顯示第一個
  加上「+N」，滑鼠移上去才會展開完整清單，確保每張卡片內部各欄位的高度
  都維持一致，不會因為某張卡片標籤特別多而讓版面看起來參差不齊。
- **獨立擴充標記（新）**：新增/編輯桌遊時，選擇「擴充」後會多一個「可
  獨立遊玩，不需要主遊戲」的勾選框。卡片與放大檢視都會標示出來，篩選
  面板的「主遊戲／擴充」也多了「可獨立遊玩的擴充」這個選項，可以直接
  篩選出這類桌遊；CSV 匯出/匯入也同步支援這個欄位。
- **上傳圖片可選取顯示範圍**：不管是新增/編輯桌遊表單，還是批次
  圖片管理裡的「上傳」，選好圖片後都會先跳出一個裁切視窗，可以直接在
  預覽框裡拖曳圖片，決定要讓卡片顯示哪個部分，拖好後按「套用」就會依
  選取的範圍存檔，不再是自動置中裁切。
- **調整已上傳圖片的大小／範圍（新）**：表單裡圖片預覽旁邊多了「調整
  大小」按鈕，批次圖片管理每一列也有同樣的按鈕，可以針對已經上傳好的
  圖片重新打開裁切工具，不需要重新選檔案上傳，就能重新選取要顯示的
  範圍。
- **放大檢視顯示完整圖片（新）**：點卡片放大檢視時，圖片區改為顯示
  完整、未裁切的原圖（會自動縮放到適合的大小，不會像小卡縮圖一樣被
  裁切），可以看到照片的全貌。
- **活動照片頁籤**：首頁「進入遊戲庫」按鈕下方多了「進入活動
  照片」的入口，進去後是一個純圖片牆（排列方式跟遊戲庫小卡一樣），可以
  用「上傳日期（新到舊／舊到新）」排序選單調整排列順序；管理者可以按
  「上傳照片」新增花絮照片並附上簡短說明；任何人點擊照片都能放大檢視，
  看到完整照片、說明文字，以及上傳日期。每張照片縮圖上也會顯示簡短說明
  跟上傳日期。管理者登入後每張照片右上角會有編輯（可以更換圖片、修改
  說明文字）跟 ✕ 刪除兩個按鈕，可以隨時調整已上傳的照片，不需要刪除
  重傳；編輯不會影響原本記錄的上傳日期。
- **類型／系列介紹依熱門程度排序**：首頁的「桌遊類型介紹」與「遊戲系列
  介紹」現在會依符合的桌遊數量由多到少排序，數量最多的自然排在最前面、
  優先顯示在預覽區。
- **今日推薦排除擴充**：自動輪替的「今日推薦」現在只會挑選主遊戲，不會
  選到擴充；如果管理者手動指定推薦桌遊，則不受此限制。
- **篩選按鈕排序管理（新）**：篩選列旁的「排序篩選」按鈕（只有管理者看得
  到）可以用上／下按鈕調整「人數、語言版本、桌遊類型、系列、發行年份、
  難度、主遊戲／擴充」這幾個篩選按鈕在工具列上的排列順序，設定後所有人
  看到的順序都會一致。
- **首頁類型／系列介紹改為預覽形式**：類型和系列數量比較多時，首頁不會
  一次全部列出，預設只顯示前 6 個，超過的部分會收在「顯示全部（共 N
  個）」按鈕後面，點下去才會展開；已展開時按鈕會變成「收合」，可以再收回
  預覽狀態。
- **每頁顯示筆數與分頁**：進入遊戲庫後，工具列可以選擇每頁顯示 20、40、
  60 筆，超過筆數會自動分頁（第 2 頁、第 3 頁⋯⋯），清單下方會出現頁碼
  按鈕，搜尋、篩選、排序、切換每頁筆數都會自動跳回第 1 頁。
- **登入機制**：右上角「使用 Google 登入」，任何人都能登入，但只有
  `ianlin1997@gmail.com`（管理者）看得到並能使用新增、編輯、刪除桌遊與
  修改標題的功能；其他人（包含登入的訪客）只能瀏覽、搜尋。
- **標題可編輯**：管理者登入後點一下網站最上方的名稱即可重新命名，
  所有人看到的都是同一個標題。
- **外觀自訂**：管理者可以在標題下方點一下編輯「副標語」（例如：週末聚會
  桌遊收藏），也可以在頁面最上方上傳一張橫幅圖片當作視覺主圖，滑鼠移到
  橫幅上會看到「上傳橫幅圖片」與移除按鈕（都只有管理者看得到）。
- **新增 / 編輯 / 刪除桌遊**：填寫**名稱**（不限語言，中文、英文、日文等都
  可以直接當作主要名稱）、**別名／其他語言名稱**（選填，方便同時收錄不同
  語言的名稱、加強搜尋）、**類型**（可多個，用逗號分隔，會依現有資料提供
  自動完成建議，減少同一類型打法不一致的問題）、**語言版本**（例如
  「繁體中文, English, 日本語」，用逗號分隔）、**主遊戲／擴充**（可選擇這
  款是主遊戲還是某款遊戲的擴充，選擴充時可再填寫所屬的主遊戲名稱，有自動
  完成建議）、**系列**（選填，例如「卡卡頌系列」，同樣有自動完成建議）、
  **發行年份**（選填）、**難度**（選填，點選 1～5 顆星，1 星最簡單、5 星
  最難）、**網址連結**（選填，例如 BGG 頁面或購買連結）、簡介、遊玩人數
  區間、遊玩時間區間，並可上傳一張圖片（會自動壓縮避免超過資料庫的單筆
  大小限制）。表單裡的「移除圖片」按鈕可以單獨清掉這張桌遊的圖片，不需要
  整筆刪除重建。
- **批次圖片管理**：工具列「清單管理」選單裡的「批次圖片管理」（只有
  管理者看得到）會列出目前所有桌遊，每一筆都有縮圖跟「上傳」/「下載」/
  「移除」按鈕，可以快速一次幫多款桌遊更換或清除圖片，不需要一筆一筆
  打開編輯視窗。清單最上面有「全選」勾選框，也可以個別勾選多筆桌遊，
  勾選後可以「批次下載」（會打包成一個 zip 檔）或「批次移除」圖片。
- **點擊放大檢視**：點擊任一張桌遊卡片（非編輯/刪除按鈕處），會彈出放大視窗
  顯示完整簡介內容、別名、發行年份、難度星等、大圖與連結，不會被卡片上
  的兩行摘要截斷。
- **搜尋與篩選（可複選）**：上方搜尋框可比對名稱、別名、類型、語言版本、
  系列、所屬主遊戲；旁邊一顆「篩選條件」按鈕點開後，會展開人數、語言版本、
  桌遊類型、系列、發行年份、難度、主遊戲／擴充這幾個獨立的分類按鈕，各自
  點開才會顯示該分類的選項（不會一次把所有條件展開），每個分類都可以
  **同時勾選多個選項**（例如同時勾選「策略」與「派對」類型，會顯示符合
  任一類型的桌遊），不同分類之間則是同時符合才會顯示，每個分類按鈕上的
  數字代表這個分類套用了幾個選項，「篩選條件」按鈕上則會顯示總共套用了
  幾個篩選條件，工具列上會出現一顆獨立的「清除篩選」按鈕（沒有套用任何
  篩選時不會顯示），點一下可一次重設全部條件。搜尋框打字時會有約 0.3 秒
  的緩衝（停止打字後才會重新搜尋），避免資料量大時每打一個字就重新整理
  畫面造成的頓頓感。
- **符合條件數量顯示**：套用搜尋或篩選後，桌遊清單上方會顯示「符合條件：
  X / Y 款桌遊」，X 是目前符合條件的數量，Y 是收藏的桌遊總數。
- **點擊標籤快速篩選**：縮圖上的「類型」「系列」「語言版本」標籤都可以
  直接點擊，會自動勾選對應的篩選條件（例如點一下「策略」類型標籤，畫面
  就會篩選出所有策略類桌遊，且會保留原本已勾選的其他條件），不用另外打開
  篩選選單選擇。
- **語言版本懸停展開**：如果一款桌遊有兩種以上的語言版本，縮圖右上角只會
  先顯示「{第一個語言} +N」的精簡標示，滑鼠移上去才會展開完整的語言清單，
  每個語言一樣可以直接點擊篩選，縮圖不會被一長串語言標籤塞滿。
- **排序**：搜尋框旁的排序選單可以切換「上傳時間（新到舊）」、「依名稱
  排序（A→Z）」、「發行年份（新到舊／舊到新）」或「依語言版本排序
  （A→Z）」，跟搜尋、篩選可以一起使用。
- **新發現！標示**：上傳未滿 24 小時的桌遊，縮圖左上角會有紅色「新發現！」
  的小標示，超過 24 小時會自動消失（畫面每幾分鐘會自動重新整理一次，不需
  重新整理頁面）。
- **清單管理（下載／匯入）**：工具列的「清單管理」按鈕會展開一個選單，
  裡面有「下載清單（CSV）」（任何人都能下載，不需要登入）與「匯入清單
  （CSV）」（只有管理者看得到，可一次上傳多筆桌遊）。匯入時是**依欄位
  名稱**對應資料（例如「名稱」「發行年份」「網址連結」等），不是依固定
  順序，所以就算欄位順序不同、或缺少某一欄，也不會匯入失敗；只要欄位
  名稱跟下載清單時用的一致就能正確對應。建議先下載一次現有清單當作
  範本，修改後再匯入。**注意：CSV 匯入不含圖片**，匯入後如果要加圖片，
  需要個別點進去編輯上傳。
- **即時同步**：管理者新增/刪除/編輯後，其他人畫面會自動更新，不用重新整理。
- **頁尾說明**：「任何人皆可瀏覽…」這段權限說明與收錄數量統計，現在放在頁面
  最下方（桌遊清單之後），標題區只保留主標題、副標語與登入按鈕，畫面更簡潔。

## 常見問題

**圖片上傳後儲存失敗？**
Firestore 單一文件上限是 1MB，程式會自動把圖片壓縮到合理大小，
但若原圖非常大仍可能超過限制，可以換一張較小的圖片再試一次。

**點了「使用 Google 登入」沒反應或跳出錯誤？**
最常見原因是還沒在 Firebase Console 的 Authentication > Sign-in method
啟用 Google 登入方式，或是網域還沒加進「已授權網域」清單，回頭檢查步驟三。

**朋友登入後說看不到新增/編輯按鈕？**
這是正常的——目前規則只讓 `ianlin1997@gmail.com` 能編輯，其他人（包含
登入的人）都只有瀏覽權限。想多加信任的人一起管理，回頭看步驟四的說明。

**忘記把 firebase-config.js 換成自己的設定會怎樣？**
畫面會顯示「連線失敗，請確認 firebase-config.js 設定是否正確」，
只要照著上面步驟一、二填好即可。
